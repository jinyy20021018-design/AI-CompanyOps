# Presentation Page Design

**Date:** 2026-04-20  
**Feature:** Auto-popup presentation page on coordinator completion  
**Status:** Approved

---

## Overview

When the CEO agent finishes coordinating all department agents, a cinematic full-screen overlay automatically appears in the CoAgent UI, presenting the complete multi-department project proposal alongside a live preview of the Engineering agent's generated frontend.

---

## User Flow

1. User submits a business request (e.g. "Design an AI fitness app")
2. CEO agent dispatches to Product, Engineering, Marketing, QA, Finance agents
3. Engineering agent produces a `app-preview.html` in addition to its architecture document
4. CEO agent consolidates all outputs and exits with code 0
5. Backend detects coordinator process exit → emits `coordinator:complete` WebSocket event with `sessionId` + artifact list
6. Frontend receives event → cinematic overlay auto-opens (no user action required)
7. User watches the intro, then browses the app preview and proposal tabs
8. User can dismiss the overlay (×) to return to the canvas, or open the shareable `/present/:sessionId` route

---

## Architecture

### Backend changes

- **`terminalRegistry.ts`** — on coordinator exit (exitCode 0), emit `coordinator:complete` event via WebSocket with:
  ```json
  { "type": "coordinator:complete", "sessionId": "...", "artifacts": [...] }
  ```
- **New endpoint** — `GET /api/sessions/:sessionId/artifacts` — returns list of artifact metadata (path, type, description) for a session
- **Static file serving** — ensure `app-preview.html` and all artifact `.md` files in the session artifacts folder are served statically

### Frontend changes

- **`App.tsx`** — listen for `coordinator:complete` WebSocket message; set `presentationSession` state to trigger overlay
- **New route** — `/present/:sessionId` — standalone shareable page; fetches artifacts via the new API endpoint
- **New component** — `PresentationOverlay.tsx` — full-screen overlay, reuses the same layout as `/present/:sessionId`
- **New component** — `PresentationPage.tsx` — the actual layout (shared between overlay and route):
  - Cinematic hero section (agent avatars, session stats, fade-in animation)
  - Split view: left = app preview iframe, right = proposal tabs
  - Footer with Export PDF and Share buttons

### Markdown rendering

- Use `react-markdown` (already likely available or easy to add) to render all `.md` artifact files
- No raw `#` symbols — all markdown rendered as styled HTML
- Dark theme styling scoped to the presentation page (purple/dark palette)

---

## UI Layout

### Phase 1 — Cinematic intro (3–4s, then auto-scrolls or user scrolls)

- Dark background (`#0a0a0f`)
- "COAGENT · AI COMPANY REPORT" label in purple uppercase
- Project title (from CEO's session name) large and bold in white
- 6 agent avatars (CEO, Product, Engineering, Marketing, QA, Finance) fade in one by one
- Session stats: agent count, tasks completed, elapsed time
- Smooth transition/scroll into Phase 2

### Phase 2 — Split view

**Left panel — App Preview**
- Browser chrome bar (traffic lights + filename `app-preview.html`)
- `<iframe>` rendering the Engineering agent's generated `app-preview.html`
- Light background inside the iframe (the generated app's own styles)

**Right panel — Proposal tabs**
- Tabs: Executive Summary | Product | Engineering | Marketing | QA | Finance
- Each tab renders the corresponding `.md` artifact via `react-markdown`
- Styled headings, bullet lists, bold text — no raw markdown syntax visible
- Scrollable independently of the left panel

**Footer**
- "Generated by CoAgent · [date]" label
- Export PDF button (opens print dialog)
- Share button (copies `/present/:sessionId` URL to clipboard)

**Close button**
- `×` top-right corner of the overlay
- Returns user to the agent canvas

## Engineering Agent Prompt Change

The Engineering agent's CLAUDE.md prompt must be updated to include:

> In addition to your architecture document, generate a self-contained `app-preview.html` file in `$COAGENT_SESSION_DIR/artifacts/`. This should be a single HTML file with inline CSS that represents the key screen(s) of the proposed app's UI. Register it with: `coagent artifact --type preview --path "$COAGENT_SESSION_DIR/artifacts/app-preview.html" --desc "App UI Preview"`

---

## Data Model

Artifacts registered by each agent:

| Agent       | Artifact file         | Type      | Tab label         |
|-------------|-----------------------|-----------|-------------------|
| CEO         | `summary.md`          | report    | Executive Summary |
| Product     | `prd.md`              | report    | Product           |
| Engineering | `architecture.md`     | report    | Engineering       |
| Engineering | `app-preview.html`    | preview   | (iframe, not tab) |
| Marketing   | `gtm.md`              | report    | Marketing         |
| QA          | `qa-review.md`        | report    | QA                |
| Finance     | `financial-model.md`  | report    | Finance           |

---

## Out of Scope

- Actual code execution or running the generated app (iframe renders static HTML only)
- PDF export implementation (print dialog is sufficient for now)
- Authentication on the `/present/:sessionId` route
- Historical session listing / session browser
